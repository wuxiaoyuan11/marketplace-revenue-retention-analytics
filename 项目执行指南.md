# 项目执行指南：E-commerce User Behavior and Revenue Analytics

这个项目的目标不是“做一个 Kaggle 分析”，而是做成一个能放进简历、能在面试里讲清楚业务价值的互联网数据分析项目。

## 一、最终作品应该长什么样

你最终需要交付 5 个东西：

1. GitHub 项目仓库：包含 README、SQL、Python 脚本、分析结论。
2. SQL 分析结果：GMV、AOV、订单量、留存、RFM、满意度等指标。
3. 可视化 Dashboard：建议用 Tableau Public 或 Power BI。
4. 一页 Insight Report：3-5 条业务发现和建议。
5. 简历项目描述：突出 SQL、业务指标、用户分层、留存和经营建议。

## 三、核心业务问题

你在项目里要回答这些问题：

1. 收入从哪里来：哪些品类、地区、月份贡献最多 GMV？
2. 用户质量如何：哪些用户是高价值用户、沉睡用户、一次性用户？
3. 留存是否健康：用户首次购买后，次月/多月复购情况如何？
4. 体验哪里有问题：物流时长、延迟、品类、地区如何影响评分？
5. 如何提升业务：怎么提升 GMV、AOV、复购率和满意度？

## 七、面试时怎么讲

你可以按这个结构讲：

1. 背景：这是一个 marketplace 电商数据集，目标是拆解收入、用户留存和体验问题。
2. 数据：包含订单、用户、商品、支付、评价、物流时间和地理位置。
3. 方法：先建宽表，再做 GMV/AOV、cohort、RFM、满意度分析。
4. 难点：支付表和商品表 join 会导致金额重复，所以订单级收入和商品级收入需要分开处理。
5. 结论：高 GMV 品类和地区不一定满意度最高，物流延迟会显著影响评分；应针对高价值用户和高潜力 cohort 做复购运营。
6. 业务建议：品类投放、物流优化、用户召回、复购激励、地区差异化运营。
